## Introduction

This project explores daily mean temperature records from a weather station in Tampa, Florida (COOP ID 80211), covering the years 1931 through early 1937. The dataset was sourced from FSU's Florida Climate Center and provides a daily time series that spans six full years of observations, making it well-suited for looking at seasonal cycles, year-over-year variation, and longer-term patterns in local temperature.

The project addresses three course requirements on top of the core visualizations:

1. **Interactive chart:** at least one plot built with `plotly` so readers can hover, zoom, and explore values directly.
2. **Accessibility:** colorblind-safe palettes (viridis family) throughout, alt text on every figure via `fig.alt`, and shape or linetype used alongside color wherever groups are distinguished.
3. **Redesign:** a before/after comparison showing a weak original chart and an improved version, with an explanation of what was wrong and how the redesign fixes it.

### Missing Data

Values of `-99.9` represent missing observations. These are filtered out before any analysis so they don't distort summary statistics or visual ranges.


```r
tpa <- tpa_raw %>%
  filter(meanTemp > -99) %>%
  mutate(
    date     = as.Date(paste(YEAR, MONTH, DAY, sep = "-")),
    month_name = factor(month.name[MONTH], levels = month.name),
    season = case_when(
      MONTH %in% c(12, 1, 2) ~ "Winter",
      MONTH %in% c(3,  4, 5) ~ "Spring",
```

## Visualizations

### Plot 1: Daily Temperature Time Series with Seasonal Shading (Interactive)

This interactive chart plots every valid daily mean temperature from 1931 through 1936. Hovering over any point reveals the exact date and temperature. The interactivity lets the reader do something a static image cannot: locate the coldest or hottest individual day across six years by hovering directly on outlier points, rather than inferring approximate values from a scale. You can also zoom in to inspect any single year or season in detail, which helps surface year-to-year differences that the full six-year span would otherwise compress.


```r
# Build hover label
tpa_hover <- tpa %>%
  mutate(
    hover_text = paste0(
      format(date, "%B %d, %Y"), "<br>",
      "Mean Temp: ", meanTemp, "\u00b0F<br>",
      "Season: ", season
    )
  )

# Base ggplot (plotly will convert it)
p1_base <- ggplot(tpa_hover, aes(
    x    = date,
    y    = meanTemp,
    color = season,
    text  = hover_text
  )) +
  geom_point(size = 1.2, alpha = 0.7) +
  scale_color_viridis_d(
    option = "turbo",
    name   = "Season",
    # Custom order so viridis hues map to intuitive seasonal colors
    guide  = guide_legend(override.aes = list(size = 3))
  ) +
  scale_x_date(date_breaks = "1 year", date_labels = "%Y") +
  scale_y_continuous(breaks = seq(30, 100, 10)) +
  labs(
    title    = "Daily Mean Temperature in Tampa, FL (1931–1936)",
    subtitle = "Each point is one day. Hover to see exact date and temperature.",
    x        = "Date",
    y        = "Mean Daily Temperature (F)",
    caption  = "Source: FSU Florida Climate Center, COOP Station 80211"
  ) +
  theme_minimal(base_size = 13) +
  theme(
    plot.title      = element_text(face = "bold", size = 15),
    plot.subtitle   = element_text(color = "gray40"),
    panel.grid.minor = element_blank(),
    legend.position  = "bottom"
  )

# Convert to plotly with custom tooltip
p1_interactive <- ggplotly(p1_base, tooltip = "text") %>%
  layout(
    legend = list(orientation = "h", y = -0.15),
    hovermode = "closest"
  )
```

## Chart Redesign: Before and After

### What Is Being Redesigned?

A common way to visualize multiple years of temperature data is with a simple bar chart showing the annual average temperature for each year. The "before" chart below represents this approach as it is often produced without much thought: a rainbow-colored bar chart with no value labels, no reference lines, and a y-axis that starts at 0 rather than near the actual data range.


```r
# Reproduce a deliberately weak version of the chart to serve as the "before"
# Problems built in intentionally:
#   1. Rainbow palette with no meaning
#   2. Y-axis starts at 0, compressing all variation into the top 10% of the chart
#   3. No data labels, so exact values are unreadable
#   4. No reference line or context
#   5. Chart title is generic and uninformative

bad_palette <- c("#FF0000","#FF7700","#FFFF00","#00FF00","#0000FF","#8B00FF")

# Filter to 1931-1936 only (1937 has a single observation and skews the palette count)
annual_avg_6 <- annual_avg %>% filter(YEAR <= 1936)

ggplot(annual_avg_6, aes(x = factor(YEAR), y = avg_temp, fill = factor(YEAR))) +
  geom_col() +
  scale_fill_manual(values = bad_palette, guide = "none") +
  scale_y_continuous(limits = c(0, 80)) +
  labs(
    title = "Temperatures",
    x     = "Year",
    y     = "Avg Temp"
  )
```

<img src="https://github.com/OwenTelis/dataviz_final_project/blob/main/figures/redesign-before-1.png" alt="A poorly designed bar chart showing annual average temperature for years 1931 to 1936. Each bar is a different bright rainbow color with no rationale for the color choice. The y-axis starts at 0 degrees F, which makes the differences between bars almost invisible since all values are near 70 degrees F. There are no value labels, no reference lines, and the title gives no useful context." style="display: block; margin: auto;" />

**What is wrong with this chart?**

- **Truncated y-axis in the wrong direction.** Starting at 0°F makes all the bars nearly the same height, because all annual averages fall between 68 and 73°F. The variation that actually exists in the data is compressed into the top 10% of the plot area, making year-to-year differences nearly invisible.
- **Rainbow palette with no information.** The six colors are arbitrary and don't encode magnitude, order, or group membership. A colorblind reader can't distinguish several of these hues, and even a reader with normal color vision gets no additional meaning from the color choices.
- **No data labels.** There is no way to read the exact value for any bar without manually tracing back to the y-axis.
- **Generic title.** "Temperatures" describes nothing. It doesn't say where, when, or what kind of temperature.
- **No reference line.** There's nothing to anchor the reader's interpretation of whether a given year was unusually warm or cool.


```r
# Overall average for the reference line
overall_mean <- mean(tpa$meanTemp)

ggplot(annual_avg_6, aes(x = factor(YEAR), y = avg_temp)) +
  geom_col(fill = "#1b4965", width = 0.6) +
  geom_hline(
    yintercept = overall_mean,
    linetype   = "dashed",
    color      = "#ef5675",
    linewidth  = 0.9
  ) +
  annotate(
    "text",
    x     = 6.45,
    y     = overall_mean + 0.25,
    label = paste0("6-yr avg: ", round(overall_mean, 1), "\u00b0F"),
    color = "#ef5675",
    size  = 3.5,
    fontface = "bold",
    hjust = 1
  ) +
  geom_text(
    aes(label = paste0(round(avg_temp, 1), "\u00b0F"), y = avg_temp + 0.15),
    color    = "white",
    vjust    = 1.5,
    fontface = "bold",
    size     = 4
  ) +
  scale_y_continuous(
    limits = c(65, 75),
    breaks = seq(65, 75, 2),
    oob    = scales::squish   # keep bars visible even at the lower limit
  ) +
  labs(
    title    = "Annual Average Temperature in Tampa, FL (1931–1936)",
    subtitle = "Y-axis zoomed to the actual data range so year-to-year differences are visible.\nDashed line = 6-year overall mean (69.8\u00b0F).",
    x        = "Year",
    y        = "Mean Annual Temperature (\u00b0F)",
    caption  = "Source: FSU Florida Climate Center, COOP Station 80211"
  ) +
  theme_minimal(base_size = 13) +
  theme(
    plot.title      = element_text(face = "bold", size = 15),
    plot.subtitle   = element_text(color = "gray40", margin = margin(b = 10)),
    panel.grid.minor = element_blank(),
    panel.grid.major.x = element_blank()
  )
```

**What the redesign fixes:**

- **Y-axis starts near the data.** By limiting the y-axis to 65–75°F, the actual variation in annual average temperature (about 4°F across six years) is now the dominant visual feature rather than a barely visible difference in bar heights.
- **Single, semantically neutral color.** All bars are the same steel blue. Color is not used to distinguish years here because the x-axis label already does that job perfectly well. A single consistent color removes the distraction of the rainbow and is fully colorblind-safe.
- **Direct value labels.** Each bar carries its exact annual average, so the reader never has to trace back to the y-axis.
- **Reference line with annotation.** The dashed line shows where the six-year average sits and is labeled directly on the chart. This gives the reader an anchor to judge which years were above or below the historical mean for this period.
- **Specific, informative title.** The title now tells the reader exactly what they're looking at: what measurement, which city, and which time period.

## Report & Discussion

### Data and Motivation

This dataset appealed to me because it's genuinely local. Tampa is only about 30 miles from St. Petersburg, so these records reflect weather patterns that would have felt familiar even today. It also covers the mid-1930s, which is historically interesting: the continental U.S. experienced some of its most extreme heat and drought events during this period (the Dust Bowl peaked from 1934 to 1936), so I was curious whether that showed up in a Florida station's record. Looking at the data, the 1934 and 1935 winters do stand out as cooler than average, even if Tampa's summers stayed almost perfectly stable across all six years.

### What the Visualizations Show Together

The four plots tell a consistent story. Summer in Tampa during the 1930s was hot and remarkably stable: July and August daily averages barely vary from one year to the next, and the ridge plot, heatmap, and seasonal boxplot all confirm this independently. Winter is where the interesting variation lives. The 1934 and 1935 winters produced the coldest individual days in the record, and that pattern shows up in every chart that breaks the data out by time period. The interactive scatter is the best tool for identifying specific extreme days, but the heatmap is probably the most efficient chart for communicating the overall pattern because it fits six years of monthly patterns into a small space without losing any resolution.

### How Visualization Principles Were Applied

- **Colorblind-safe palettes throughout.** Every chart in this report uses a viridis family option (viridis, magma, plasma, or turbo). None of them rely on red/green distinctions.
- **Shape as a redundant encoding in Plot 4.** In the seasonal boxplot, year is encoded with both color and point shape, so the chart is readable even in grayscale or by a reader who can't distinguish the viridis hues.
- **Alt text on every figure.** Each `{r}` chunk includes a `fig.alt` option with a full description of what the chart shows, including approximate values, so the visualization is accessible to screen reader users.
- **Direct labels over legends wherever possible.** The heatmap tiles carry their exact values directly, eliminating the need to match colors to a continuous scale. The redesign bar chart carries value labels on each bar for the same reason.
- **Interactivity with a clear purpose.** The interactive scatter (Plot 1) uses `plotly` so readers can hover to get exact values for outlier days. A static version of this chart would require the reader to estimate temperature from position, which is imprecise; hover tooltips make the exact value instantly available.

---

## Conclusion

Six years of Tampa temperature data is a narrow window, but it's enough to show a few clear patterns: summer is stable, winter is variable, and the mid-1930s winters were measurably colder than the early part of the record. The most useful chart for communicating this to a general audience is probably the heatmap (Plot 3), because it compresses the full picture into a single glance. The interactive scatter (Plot 1) is the most useful for a reader who wants to explore specific dates or outlier events. The redesign exercise confirmed something that comes up repeatedly in this course: y-axis choices matter enormously. Starting a bar chart at 0 when all the data sits between 68 and 73°F doesn't make the chart more honest, it just makes it unreadable.
